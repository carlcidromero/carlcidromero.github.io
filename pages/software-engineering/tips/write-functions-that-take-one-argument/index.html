Write functions that take one argument

Functions that take one argument:

  • Are known as unary functions.
  • Have an arity of 'one'.
  • Help keep functions 'pure'.
  • Encourage focused, concise, and modular code.

Examples in JavaScript


  /**
    * @param {string} value
    */
  const operateOnValue = (value) => secondOperation(firstOperation(value));
  • I prefer unary over variadic functions—especially when currying, partial application, and thunks join the party: 

  /**
   * @param {(value: string) => string} firstOperation
   */
  const curried =
    (firstOperation) =>
    /**
     * @param {(value: string) => string} secondOperation
     */
    (secondOperation) =>
    /**
     * @param {string} value
     */
    (value) => secondOperation(firstOperation(value));
  
  const partiallyApplied = curried(someOperation)(anotherOperation);
  const delayedEvaluation = partiallyApplied('hello');
  const thunk = () => partiallyApplied('hello');
  • Curries, delayed evaluation, partial application, and thunks will be covered in detail later. For now, just know that they are techniques for a declarative, functional style of programming. At the time of this writing, however, imperative programming is the norm. So, to avoid confusion during collaboration, I typically write pure unary functions by way of a 'config' object in team environments: 

  /**
   * @param {{foo: string, bar: number, baz: boolean}} config
   */
  const operateOnValues = (config) => {
    const { foo, bar, baz } = config;
    const result = {};
    result.foo = stringOperation(foo);
    result.bar = numberOperation(bar);
    result.baz = predicate(baz);
    return result;
  }
  • These are more developer-friendly than variadic functions that require multiple arguments passed in a specific order.
