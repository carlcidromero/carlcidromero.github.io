Write functions that take one argument

Functions that take one argument:

  • May also be referred to as unary or monadic functions.
  • Encourage focused, concise, and modular code.

Examples in JavaScript

A unary function that takes a primitive argument

  • I rarely write these because their arity can too easily increase while maintaining a codebase. 
          
  /**
    * @param {string} value
    */
  const operateOnValue = (value) => {
    const result = someOperation(value);
    return result;
  }

A unary function that takes a non-primitive argument

  • I prefer to write these because they can easily remain unary.
  • They are more developer-friendly than functions that require multiple arguments passed in a specific order. 
          
  /**
   * @param {{foo: string, bar: number, baz: boolean}} values
   */
  const operateOnValues = (values) => {
    const { foo, bar, baz } = values;
    const result = {};
    result.foo = stringOperation(foo);
    result.bar = numberOperation(bar);
    result.baz = booleanOperation(baz);
    return result;
  }
