01: MVP Static GitHub Page

November 8, 2024

This blog starts out very basic. You can do the same if you have internet access by creating a GitHub account, creating a repository, setting it up as a GitHub Page, and creating a homepage with an HTML file.

In this post, I will summarize the technologies involved in this current version of the blog, the design decisions I made, a little bit about interesting parts of its implementation, and what enhancements will be made next.

Tech Stack

  • CSS: Styling website layout and elements.
  • GitHub: Code version control and website hosting.
  • HTML: Website structure and content.
  • JSON-LD: Enriching search engine result pages and search engine optimization.

Design

The design is very minimal — HTML is responsive out-of-the-box, so I add as little CSS as possible to keep it that way. I am literally doing just enough to share text, so for now, the only changes to the document layout are related to readability.

Implementation

Key points of this website's implementation are in the subheadings below. For those that want the full picture, feel free to check out the MVP source code.

CSS On This Site

The entire page is a flex box to simplify layout adjustments. I opted to take a mobile-first approach, too, so I am keeping the content at max-width: 500px.

What's Next?

There are a few enhancements I would like to make — web components for redundant HTML elements on this site (like the navbar), syntax highlighting for the code snippets, a dark mode toggle... the list could go on. I plan on adding these enhancements eventually, but I will only focus on just one for now. That will be the most useful and meaningful enhancement that comes to mind, which is to add JavaScript and use GitHub Gist as a content management system. With this in place, I can write blog posts in Markdown from my phone instead of writing them in HTML. As a result, this site should become easier to maintain, since the content will be dynamically rendered on each page, and should be a nice example of consuming a REST API for readers.

If you would like to suggest an enhancement for or report a bug on this site, feel free to create an issue. You will need a GitHub account to do so, and if you are an aspiring or new software engineer, this is a good process to become familiar with!
